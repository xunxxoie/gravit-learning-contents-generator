## 학습 콘텐츠 표기·용어 스타일 규칙

### 적용 대상 필드

`lesson.sql`의 아래 문자열 리터럴에 적용한다.

- **lesson_staging.title**
- **problem_staging.instruction** / **content**
- **option_staging.content** / **explanation**
- **answer_staging.content** / **explanation**

SQL 키워드·컬럼명·ID·**label** 값은 해당 없음.

---

### S1. 한국어 맞춤법·띄어쓰기

- 표준 맞춤법을 지켜라.
  - **데이타** → **데이터**, **역활** → **역할**, **되요** → **돼요**
  - **틀리다**(잘못 선택됨)와 **다르다**(상이함) 구분
- 명백한 띄어쓰기 오류를 피하라.
  - **할수있다** → **할 수 있다**
  - **프로세스간** → **프로세스 간**
- 조사·어미 선택을 정확히 하라.
  - **~로써**(수단) / **~로서**(자격) 구분
  - **~에**와 **~에서**의 혼동 주의

### S2. CS 도메인 외래어 표기

표준 외래어 표기법을 따른다. 자주 틀리는 것:

| 금지 | 올바른 표기 |
|---|---|
| 해쉬 | 해시 |
| 캐쉬 | 캐시 |
| 메세지 | 메시지 |
| 쓰레드 | 스레드 |
| 알고리듬 | 알고리즘 |
| 데이타 | 데이터 |
| 네트웍 | 네트워크 |
| 서브넷트 | 서브넷 |
| 엑세스 | 액세스 |
| 컨텐츠 | 콘텐츠 |
| 비지니스 | 비즈니스 |
| 어플리케이션 | 애플리케이션 |

위 표는 고빈도 예시다. 확신이 없는 외래어는 표준 표기를 따른다.

### S3. 용어 일관성 (한 레슨 내)

- **같은 개념을 한 레슨 안에서 혼용하지 마라.** 예: 한 문제에서 **JavaScript**, 다른 문제에서 **자바스크립트** → 둘 중 하나로 통일한다.
- 영문/한글 표기는 레슨 단위로 하나를 고른다. 레슨마다 다른 선택은 무방하나, 한 레슨 내부에서는 고정한다.
- 약어와 풀네임은 최초 등장 시 **풀네임(약어)** 형태를 권장한다. 강제는 아니지만 선택한 형태는 유지한다.
- 6문제를 한 번에 쓰는 시점에 전체 가시성이 있으므로, 쓰기 직후 **전체를 훑어 일관성을 점검**한다.

---

### S4. CS 용어 적절성

- 문맥에 맞는 용어를 선택하라.
  - 잘못된 예: "프로세스가 스레드를 상속한다" (상속은 OOP 개념에 쓰는 말)
- 은어·구어를 쓰지 마라.
  - 금지: **디비**, **자스**, **셤** 등 → **데이터베이스**, **JavaScript**, **시험**
- 개념적으로 부정확한 표현을 피하라. (예: "TCP는 UDP보다 항상 빠르다")

---

### S5. 타이핑·스펠링

- 자음과 모음 누락·중복을 확인하라. (예: **구현해한다**, **알고리즘은은**)
- 영문 고유명사의 철자를 정확히 쓰라. (예: **JavaScript**, **PostgreSQL**, **Kubernetes**)

---

### S6. 숫자·단위 표기

계산 문제·수치가 들어가는 본문/선지에 적용한다.

- 네 자리 이상 정수는 천 단위로 콤마를 찍는다. **1048576** → **1,048,576**. (2의 거듭제곱을 강조할 때는 **2²⁰(1,048,576)**처럼 병기해도 된다.)
- 용량과 속도의 단위를 구분한다. 메모리·디스크 용량은 바이트 단위(**KB·MB·GB**), 전송 속도는 비트 단위(**Kbps·Mbps**)다. 한 맥락에서 비트(**b**)와 바이트(**B**)를 섞지 마라.
- 수치와 단위 사이 띄어쓰기는 한 레슨 안에서 하나로 통일한다(**4KB** 또는 **4 KB** 중 하나). 단위 약어는 대문자(**KB·MB**)를 권장한다.
- 진법은 접두로 밝힌다: 16진수 **0x1F**, 2진수 **0b1010**. **1F(16진수)**처럼 풀어 써도 된다.

### 주의

- 본문에 코드 블록(** **`**java ... **`** ** 등)을 포함할 때, 코드 블록 내부의 키워드·식별자는 맞춤법 규칙의 대상이 아니다.
- 스타일 규칙은 SQL 이스케이프 전의 원문(작은따옴표를 **''**로 바꾸기 전 텍스트) 기준으로 적용한다.
- 본문(content)에는 마크다운(코드 블록·표·여러 줄)을 쓸 수 있다. 이스케이프 규칙은 `learning-content-sql-template.md`의 "문자열 이스케이프" 절을 따른다.
